# Project Vision

Build a complete parental control ecosystem for Windows managed from a mobile application.

> This roadmap is a living document.
> It may change as the team discovers better ideas during development.

Current Active Plan

Version 1.0.1

Windows-first Parent/Child role selection with approval-based pairing.

docs/VERSION_1_0_1_PLAN.md

# Development Philosophy

Priority Order

1. Stable
2. Secure
3. Easy to Extend
4. Beautiful UI
5. Advanced Features

We prioritize architecture over feature count.

---

# Phase 1 - Version 1.0.1 Windows Foundation

Goal

Build a working Windows-first MVP.

Expected Result

- User can choose Parent or Child mode in the Windows client
- Parent can register and login
- Child can display a connection code
- Parent can request pairing with the code
- Child can approve or reject the request
- Approved child device can be managed by Parent
- Windows Service protects the child device after approval

Status

Active

# Idea Parking Lot

Every new idea should be recorded here first.

Example

- Reward system
- Emergency unlock

- Parent notification
- Focus timer

Recording an idea does NOT mean it will be implemented.

---

# Decision Rule

A new feature should only be implemented if

- It does not break current architecture.
- It does not delay the current milestone.
- The team agrees.
- Documentation is updated.
