# ROADMAP.md

# Project Vision

Build a complete parental control ecosystem for Windows managed from a mobile application.

> This roadmap is a living document.
> It may change as the team discovers better ideas during development.

---

# Development Philosophy

Priority Order

1. Stable
2. Secure
3. Easy to Extend
4. Beautiful UI
5. Advanced Features

We prioritize architecture over feature count.

---

# Phase 1 - Foundation

Goal

Build a working MVP.

Expected Result

- Parent can login
- Parent can pair a device
- Windows Agent connects successfully
- Parent changes mode
- Agent applies mode

Status

In Progress

---

Possible Improvements

- Better error handling
- Performance improvements

This phase may change.

Possible Directions

---

# Idea Parking Lot

Every new idea should be recorded here first.

Example

- Reward system
- Emergency unlock
- Temporary permission
- Parent notification
- Focus timer

Recording an idea does NOT mean it will be implemented.

---

# Decision Rule

A new feature should only be implemented if

- It does not break current architecture.
- It does not delay the current milestone.
- The team agrees.
- Documentation is updated.

---

# Living Document

This roadmap is expected to change.

Changing the roadmap is normal.

The current sprint always has higher priority than future ideas.
